# CLAUDE.md - Cave a Vin

Quick-start guide for Claude Code agents working on this project.
For detailed conventions, checklists and testing procedures, see `AGENT_GUIDELINES.md`.

## Development commands

```bash
# Dev server (Flask, port 5000)
FLASK_DEBUG=1 python run.py

# Docker build & run (Gunicorn, port 8000)
docker build --network=host -t macave:debug .
docker run --rm -it --name macave -p 8000:8000 \
  -v $(pwd)/data:/data \
  -e DATABASE_URL=sqlite:////data/wine.db \
  -e SECRET_KEY=DevSecret \
  -e DEFAULT_ADMIN_PASSWORD=changeme \
  macave:debug

# Docker Compose (app + scheduler)
docker-compose up -d

# Production entry point
gunicorn wsgi:app
```

There is no test suite yet. Verify changes by building the Docker image and testing manually in the browser.

## Architecture

- **Flask factory** in `app/__init__.py` (`create_app()`), config in `config.py`.
- **WSGI entry**: `wsgi.py` (Gunicorn) / `run.py` (dev server).
- **Blueprints** in `app/blueprints/` — one file per domain (wines, cellars, categories, auth, api, search, admin, etc.).
- **Models** in `app/models.py` — SQLAlchemy, SQLite by default (Postgres/MySQL supported via `DATABASE_URL`).
- **Services** in `services/` — `wine_info_service.py` handles OpenAI enrichment.
- **Templates** in `templates/` — Jinja2, all extend `base.html`. Reusable macros in `_macros.html`.
- **Static** in `static/` — Bootstrap 5.3, vanilla JS.
- **Scheduler** — separate container (`Dockerfile.scheduler`), runs weekly reports and cleanup tasks.

## Database migrations

No Alembic. Migrations are **manual and idempotent** in `app/database_init.py`:

- New columns: add to model in `app/models.py`, then add an idempotent `ALTER TABLE` block in `apply_schema_updates()`.
- New tables: add model in `app/models.py`; `db.create_all()` handles creation. Seed data goes in `initialize_database()`.
- Always check table/column existence before altering. Use `nullable=True` for new columns.

## Key conventions

- **Sub-account system**: `User.owner_id` returns the effective owner (parent or self). Always filter resources by `owner_id`, never `current_user.id` directly.
- **API auth**: Bearer tokens via `@api_token_required` decorator (`app/utils/decorators.py`). Access user with `g.api_user`.
- **Web auth**: Flask-Login, `@login_required`. CSRF enabled (API blueprint is exempt).
- **Code style**: PEP 8, type hints on public functions, Google-style docstrings. Imports grouped: stdlib / third-party / local.
- **Frontend**: Vanilla JS + Bootstrap 5.3, no frameworks. Prefer Bootstrap utility classes over custom CSS.
- **Language**: Code and comments in French where they already are. Commit messages in English (conventional commits).

## Versioning

- **Conventional commits** — `feat:`, `fix:`, `docs:`, `refactor:`, etc.
- **Changelog** generated by [git-cliff](https://git-cliff.org/) (`cliff.toml`).
- **Semantic versioning** — `APP_VERSION` set via env var or Docker build arg.
- CI workflows: `.github/workflows/changelog.yml` (changelog) and `.github/workflows/docker.yml` (Docker image).
